vue.js store模式学习

发表于 2019-01-14

vue.js store模式学习

说在前面

我理解的所谓 store模式 就是vuex在小型项目或者一个小的模块中的简化版。Vuex 是一个专为 Vue.js 应用程序开发的状态管理模式，而store模式同样可以实现小型模块的状态管理。

一个简单的store模式

需要注意，所有 store 中 state 的改变，都放置在 store 自身的 action 中去管理。这种集中式状态管理能够被更容易地理解哪种类型的 mutation 将会发生，以及它们是如何被触发。



接着我们继续延伸约定，组件不允许直接修改属于 store 实例的 state，而应执行 action 来分发 (dispatch) 事件通知 store 去改变。这样约定的好处是，我们能够记录所有 store 中发生的 state 改变，同时实现能做到记录变更 (mutation)、保存状态快照、历史回滚/时光旅行的先进的调试工具。

实践

在实际项目中，组件之间的数据传递是通过触发组件的属性(prop)和事件($emit)。
实践中多个单独的组件可以引用同一个状态管理对象来分享数据。（在石榴计hnweiquan分支中维权功能中的提交维权信息就是使用的这种模式）
